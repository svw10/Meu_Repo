# PROTOCOLO DE APRENDIZADO CONTÍNUO (LEARNING PROTOCOL)
> **OBJETIVO:** Garantir que nenhum erro seja cometido duas vezes e que toda solução genial seja imortalizada.
> **QUEM USA:** Todos os Agentes (principalmente ZETA e ETA).

---

## 1. O CICLO DE APRENDIZADO
O aprendizado não é automático; ele deve ser **provocado**. Sempre que você terminar uma tarefa complexa, execute este ciclo:

1.  **EXECUÇÃO:** O Agente GAMMA ou ETA resolve o problema.
2.  **ANÁLISE:** O resultado foi melhor que o esperado? Ou houve um erro grave antes de acertar?
3.  **REGISTRO:** Se a resposta for "SIM", você deve gravar o aprendizado.

---

## 2. ONDE GRAVAR (MEMÓRIA DE LONGO PRAZO)
Dependendo do tipo de aprendizado, salve no local correto:

| TIPO DE APRENDIZADO | ONDE SALVAR | FORMATO |
| :--- | :--- | :--- |
| **Correção de Bug Recorrente** | `Minhas Skills\solucionando-erros\resources\known_issues.md` | `[ERRO] -> [SOLUÇÃO]` |
| **Novo Padrão de Código** | `Snippets\` (Crie um arquivo novo) | Código reutilizável |
| **Melhoria de Processo** | `Minhas Skills\gerenciando-memoria\MEMORY_LOG.md` | Texto livre (Lição Aprendida) |
| **Regra Proibitiva** | `Minhas_Rules\stack-omega.policy` | Adicionar à lista de "Proibições" |

---

## 3. FORMATO DE REGISTRO DE LIÇÃO
Ao escrever no `MEMORY_LOG.md`, use sempre este template para facilitar a leitura futura:

```markdown
### [DATA] - [TIPO: BUG/FEATURE/OPTIMIZATION]
**Contexto:** O que estávamos tentando fazer?
**O Problema:** O que deu errado ou estava difícil?
**A Descoberta:** Qual foi o "pulo do gato"?
**Ação Futura:** O que mudamos nos Agentes para não esquecer isso?
